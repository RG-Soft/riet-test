Справочник "Parcels"

Справочник входит в подсистемы Import / export и Local distribution.

Справочник предназначен для отражения грузовых мест.
Parcel (его еще часто называют "Piece") обычно соответствует коробке с однородными товарами.
Товары (items, invoice lines) и дополнительная информация (такая как количество, серийный номер и т. д) указываются в табличной части Items.

Для целей импортной транспортировки Parcels объединяются в DOC (Delivery orders consolidation).
Parcel вместе с DOC может быть загружен автоматически из pdf-файла с помощью обработки DOC loading.

В экспортном модуле Parcels объединяются в Export request.

Для целей локальной доставки Parcels объединяют в документ Trip, в котором указано место и предполагаемые даты доставки грузов.

Защита от изменения

Parcel закрывается от редактирования:

  • после перевода DOC на этапе Booked (при этом DOC должен быть проведен)
  • после перевода Export request на этап Accepted by specialist (для пользователей с ролью tracker)
    или после перевода Export reqeust на этап Local ATA или Consignee GL received (для остальных пользователей)
  • после выгрузки документа Trip

Реквизиты шапки

  • No. (строка)
    Обязательный для заполнения реквизит.
    Система контролирует уникальность номера.
  • Final (булево) - признак того, что Parcel проверен, непроверенный Parcel является "черновиком".
    Нельзя пометить проверенным отмененный Parcel.
  • Cancelled (булево) - признак того, что Parcel отменен.
    Нельзя отменить проверенный Parcel.
  • For deletion (булево) - признак того, что Parcel не просто отменен, а должен быть удален из системы.
    При пометке на удаление автоматически ставится галочка Canceled.
    При снятии пометки на удаление, галочка Canceled снимается.
  • DO no. (строка)
    Скрывается для экспортных парселей.
    Обязательный для заполнения для импортных парселей.
  • Export request (документ Export request) - Export request к которому привязан парсель.
    Скрывается для импортных и локальных парселей.
    Не редактируется.
  • Packing type (строка длины 150)
    Обязательный для заполнения реквизит.
  • Length, Width, Height (числа) - размеры
    Обязательные для заполнения реквизиты.
  • DIMs UOM (справочник UOMs) - единица измерения размеров.
    Обязательный для заполнения реквизит.
    Единица измерения не должна быть помечена на удаление.
    В качетсве стандартой единицы измерения должен быть указан сантиметр.
  • Length, cm., Width, cm., Height, cm. (числа) - размеры, приведенные к сантиметрам (чтобы потом не складывать дюймы с футами), рассчитываются автоматически.
  • Cubic meters (число 10,4) - объем в кубических метрах, рассчитывается автоматически.
  • Net weight, Gross weight (числа) - вес нетто и брутто.
    Обязательные для заполнения реквизиты.
  • Weight UOM (справочник UOMs) - единица измерения веса.
    Обязательный для заполнения реквизит.
    Единица измерения не должна быть помечена на удаление.
    В качестве стандартной единицы измерения должен быть указан килограмм.
  • Net weight, kg., Gross weight, kg. (числа) - веса, приведенные к килограммам (чтобы потом не складывать фунты с тоннами), рассчитываются автоматически.
  • Volume weight (число) - объемный вес, рассчитывается автоматически.
    Некоторые предметы весят мало, но они объемны.
    В этом случае цена отправок определяется не по весу, а в соответствии с их «объемным весом» (volume weght).
    Следует рассчитывать объемный вес каждой отправки и сравнивать его с реальным весом.
    В случае, если объемный вес превышает фактический, тариф на доставку определяется по объемному весу - это и есть chargeable weight.
    Формула для расчета объемного веса: Объемный вес (кг) = Длинна (см) х Ширина (см) х Высота (см) / 6000.
  • Chargeable weight (число) - вес, по которому потом делятся расходы на транспортировку, рассчитывается автоматически.
    Определяется как наибольшее из Вес Брутто и Вес Объемный.
    Если размеры не заданы, то объемный вес = 0, соответственно вес Брутто будет больше.
  • PO list (строка) - список PO, к которым относятся Invoice lines таблицы Goods, заполняется автоматически.
  • Invoice list (строка) - список Invoices, к которым относятся Goods, заполняется автоматически.
  • Created by (справочник Users) - заполняется автоматически пользователем, который создал парсель.
  • Creation datе (дата и время) - заполняется автоматически датой создания документа.
  • Modified by (справочник Users) - заполняется автоматически пользователем, который внес последние изменения в документ.
  • Modification date (дата и время) - заполняется автоматически датой последнего интерактивного изменения документа.

Реквизиты Local distribution

  • Warehouse from (справочник Warehouses) - склад отправления.
    Обязательный для заполнения, если стоит признак Local only.
  • Warehouse to (справочник Warehouses) - склад прибытия.
    Обязательный для заполнения, если стоит признак Local only.
  • Urgency (перечисление Urgencies) - срочность груза.
    Заполняется автоматически из DOC, к которому относится Parcel.
  • Hazard class (перечисление Urgencies) - класс опасности.
  • Description (строка) - краткое описание содержимого грузового места.
  • Local only (булево) - признак того, что Parcel участвует только в локальной транспортировке.
  • Repacked (булево) - признак того, что Parcel переупакован перед локальной транспортировкой.

Реквизиты табличной части "Items"

Список Items не может быть пустым, если в Parcel стоит признак Final.

  • Item (справочник Items) - товар
    Изначально в печатной форме Parcel этого реквизита нет.
    Есть только строковые значения PO и PO Line, по которым в документе DOC можно определить Goods.
    В дальнейшем этот "фиктивный" реквизит необходим для того, чтобы распределить стоимость перевозки, которая ложится на Parcel, по Goods.
    Обятательный для заполнения реквизит.
    Не должен быть помечен на удаление.
    Invoice не должен быть отменен.
    Импортные товары должны относиться к одной операционной локации.
    Значения реквизитов в строке должны совпадать с соответствующими реквизитами справочника Goods.
    При интерактивном выборе - система автоматически накладывает необходимые фильтры.
  • PO no. (строка)
    Скрывается для экспортных парселей.
  • PO line no. (число)
    Скрывается для экспортных парселей.
  • Receiver (строка)
    Скрывается для экспортных парселей.
  • S/N (строка) - серийный номер.
  • Qty (число) - количество.
    Используется для распределении веса всего Parcel по строкам.
    Мы имеем право распределять по количеству, потому что, как правило, в Parcel везут однородные товары.
    Обязательный для заполнения реквизит для импортных и экспортных Parcels.
    Количество в парселях не должно превышать количество, указанное в товаре.
  • UOM (справочник UOMs) - единица измерения количества.
    Обязательный для заполнения реквизит для импортных и экспортных Parcels.
    Единица измерения должна совпадать с единицей измерения в товаре.
  • Gross weight (числа) - расчитываемый автоматически вес брутто каждого товара.
    Вес в шапке распределяется автоматически пропорционально количеству, указанному в строках табличной части.
  • Chargeable weight (число) - расчитываемый автоматически вес к оплате каждого товара.
    Вес к оплате из шапки Parcel распределяется автоматически пропорционально количеству, указанному в строках табличной части.
