Справочник "Items"

В справочнике Items хранится перечень товаров с их описанием, количеством, ценой и т. п.

Справочник отностися к нескольким подсистемам: Import / Export, Services, Local distribution.

В подсистеме Import товары обязательно привязываются к документу Import invoice и могут быть отредактированы только из формы этого документа, но не из формы товара.

В подсистеме Export товары также должны быть обязательно привязаны к документу, он назвается Export request.
Однако редактирование товаров происходит не в форме документа, а в отдельной форме элемента справочника Items.
Товары нельзя изменять, если в указанном Export request заполнена дата ATA FD.

В подсистеме Local distribution локальные товары не привязываются к документам и также редактируются в форме элемента справочника.
Товары нельзя изменять, если они указанных в проверенных Parcels, у которых стоит признак Final.

Структура

  • Code (строка) - внутренний порядковый номер товара. Присваивается автоматически.
  • Name (строка) - автоматически формируемое представления товара для удобства идентификации в документах и отчетах.
    Для импортных товаров используется представление в виде [Номер документа] - [Номер строки документа].
    Для экспортных товаров используется представление в виде [Номер документа] - [Код товара].
    Для локальных товаров в качестве представления используется только код.
  • For deletion (булево) - признак того, что товар должен быть удален из системы.
  • Invoice (документ Invoice) - инвойс, к которому привязан товар.
    Реквизит Export request должен быть пустым.
    Скрывается для локальных и экспортных товаров.
  • Invoice line no. (число) - порядковый номер товара в инвойсе.
    Скрывается для локальных и экспортных товаров.
    В системе не может быть двух не помеченных на удаление товаров с одинаковым инвойсом и номером строки.
  • Export request (документ Export request) - заявка на экспорт, к которой привязан товар.
    Реквизит Invoice должен быть пустым.
    Скрывается для импортных и локальных товаров.
  • PO no. (строка) - номер заявки на закупку.
    Скрывается для экспортных товаров.
  • PO line (справочник PO lines) - строка заявки на закупку.
    Скрывается для экспортных товаров.
    Реквизит должен быть заполнен для GOLD Invoice.
    PO не может быть помечена на удаление.
    Строка заявки на закупку не может быть помечена на удаление.
    Строка должна относиться к PO, номер которой указан в реквизите PO no.
    Компания заявки на закупку должна совпадать с реквизитом Sold-to в товаре.
    Part no. строки заявки на закупку должен совпадать с Part no. в товаре.
    UOM строки заявки на закупку должна совпадать с UOM в товаре.
    Валюта строки заявки на закупку должна совпадать с валютой в товаре.
    ERP treatment строки заявки на закупку должен совпадать с ERP treatment в товаре.
    AU строки заявки на закупку должен совпадать с AU в товаре.
    Activity строки заявки на закупку должен совпадать с Activity в товаре.
    При выборе из формы система автоматически накладывает фильтры по Part no., Description, UOM и др.
    При изменении этого реквизита могут перезаполняться другие реквизиты. См. ниже.
  • Part no. (строка) - артикул товара.
    Обязательный для заполнения для импортных и экспортных товаров.
    Заполняется автоматически при выборе PO line.
  • S/N (строка) - серийный номер товара.
  • RAN (строка) - обязателен для заполнения для тех Export requests, у которых в реквизите Export purpose указан RAN.
  • Description (строка) - описание товара.
    Обязательный для заполнения.
    Заполняется автоматически при выборе PO line.
  • Catalog (справочник Catalog) - ссылка на товар из каталога Шлюмберже.
    Не всегда товары в Инвойсах проходят по каталогу, в этом случае поле Каталог не будет заполнено.
    Запоняется автоматически при выборе PO line.
  • Qty (число) - количество.
    Обязательный для заполнения реквизит.
  • Qty UOM (справочник UOMs) - единица измерения количества.
    Обязательный для заполнения реквизит.
    Заполняется автоматически при выборе PO line.
    Если экспортный товар выгружается в TMS, то проверяется, что указанная единица измерения есть в TMS.
  • Net weight (число)
  • Net weight UOM (справочник UOMs) - единица измерения веса нетто.
    Обязательный для заполнения реквизит, если указан вес нетто.
    Если экспортный товар выгружается в TMS, то проверяется, что указанная единица измерения есть в TMS.
  • Unit Price (число)
    Обязательный для заполнения реквизит.
    Заполняется автоматически при выборе PO line.
  • Currency (справочник Currencies)
    Обязательный для заполнения для импортных и экспортных товаров.
  • Total price (число)
    Обязательный для заполнения.
    Total price должен равняться Price * Qty.
  • Import reference (строка) - номер документа-основания.
    Для импортных GOLD инвойсов в этом поле хранится номер Receiver'а.
    В случае проблем с грузом (вложением / качеством) все претензии должны идти с указанием Reсeiver.
    Для non-GOLD или экспортных инвойсов в этом реквизите может быть номер какого-нибудь другого документа.
    Обязательный для заполнения для импортных и экспортных товаров.
  • Parent company (справочник Parent companies (RCA) - юридическое лицо.
    Скрывается для экспортных товаров.
    Обязательный для заполнения реквизит.
  • AU (справочник Accounting units)
    В дальнейшем этот AU будет использоваться для загрузки всех расходов в Lawson.
    Обязательный для заполнения реквизит.
    Не разрешено использовать AU "DUMMY".
    Заполняется автоматически при выборе PO line.
  • Activity (строка) - Activity наравне с AU используется при проведении расходов в Lawson.
    Activity используется только для 500-х счетов, соответственно у Assets и Inventory (managment type) Activity будет пустым.
    Обязательный для заполнения реквизит для определенных ERP treatments.
    Заполняется автоматически при выборе PO line.
  • ERP treatment (перечисление ERP Accounting Treatments) - управленческая классификация товара.
    Обязательный для заполнения реквизит.
    Заполняется автоматически при выборе PO line.
  • Country of origin (строка или справочник Countries / HUBs) - та страна, которая указывается в счетах.
    Как правило она не является страной происхождения товара. Это - страна компании-производителя (например, для телевизоров Sony, это всегда будет Япония, хотя фактически все они сделаны в Малазии).
    Для таможенного оформления нужна именно страна, где товар был непосредственно произведен, поэтому полностью полагаться на этот реквизит нельзя, и именно поэтому он сделан текстовым.
    Обязательный для заполнения для импортных и экспортных товаров.
    Если экспортный товар выгружает в TMS, то проверяется, что указанная страна действительно есть в TMS.
  • HTC (строка) - используется для сопоставления международного кода ТНВЭД и российского кода, который был присвоен на таможне.
  • Prior CCD (строка) - используется для определения первоначальной импортной декларации при экспорте товаров.
  • PSA (справочник PSA contracts) - если в PSA заполнен Segment, то проверяется совпадение с Segment'ом AU.
  • Permanent / temporary -определяет относится товар к временному ввозу или нет.
  • Permits required - признак устанавливается, если нужны дополнительные разрешения для ввоза товара. 
  • Final - для импортных товаров проставляется автоматически при проведении инвойса, для экспортных товаров признак устанавливается вручную.
