Справочник Process levels

Process level - обособленная группа специалистов со своими уникальными бизнес-процессами.
Process level похож на RLC или Corporation в TMS, однако, например, Казахстан и Узбекистан - это один RLC (и один Corporation), а в RIET - это разные Process levels.

Этот справочник используется во всех подсистемах.

Добавлять элементы в справочник могут только разработчики.

Реквизиты

  • Code (текст)
    Обязательный для заполнения реквизит.
    Контролируется уникальность.
  • Name (текст)
    Обязательный для заполнения реквизит.
  • Country (справочник Countries)
    Обязательный для заполнения реквизит.
  • Corporation (справочник Corporations) - именно Corporation, а не Process level используется в отчетах.
    Обязательный для заполнения реквизит.
  • Leave documents in desktop until all items are in customs files (булево) - признак того, что импортные поставки и экспортные реквесты нужно оставлять в рабочем столе до тех пор, пока все товары не войдут в декларации.
  • E-mails for temporary items notifications (текст) - адреса, на которые будут приходить уведомления об истечении срока временного импорта или экспорта.
    Можно перечислить несколько через запятую.
  • From TMS IDs list (текст) - поле, в котором автоматически перечисляются все TMS ID, которые соответсвуют данному Process level. Это нужно для удобства отображения TMS ID в списке Process levels.

Таблица From TMS IDs

В этой таблице указываются TMS ID, по которым система будет определять Process level.
Таблица нужна потому, что для одному Process level может соответствовать несколько TMS Corporations. Например Corporations RU West и RU Central соответствуют одному Process level RUWE.

  • From TMS ID (текст)
    Контролируется, что указанный TMS ID не встречается в других Process levels.
    Система автоматически удаляет пустые строки и дубли.
