Подсистема "Services"

Назначение

Подсистема "Services" предназначена для

  • отражения услуг, связанных с импортом или экспортом товаров;
  • распределения стоимости услуг по AUs;
  • учета счетов поставщиков услуг;
  • выгрузки данных об оказанных услугах в Lawson.

Структура подсистемы

Перечень всех услуг, которые могут быть оказаны поставщиками хранится в справочнике "Services".

Оказанная услуга отражается документом "Service", который создается из документа "Shipment".

При заведении документа "Service" цена услуги может быть подставлена автоматически, если в системе предварительно завести необходимые документы "Price list of services".

В документе "Service" отражается только факт оказания услуги, стоимость услуги не распределяется по товарам.
Для распределения стоимости услуг по товарам предназначен документ "Services costs allocation".
Следует помнить, что аллокацию стоимости можно произвести только после закрытия поставки, то есть только после того как введен документ "Invoice lines matching" из подсистемы "Shipment closing".
Перечень несаллокированных услуг выводится в отчете "Unallocated services".

Для отражения счета поставщика предназначен документ "Agent invoice".
Перечень саллокированных услуг, по которым еще не получен счет, отображается в отчете "Not receive agent invoices".

Документ "Agent invoice" также позволяет сформировать форму для выгрузки данных в Lawson.
Детальная информация по стоимости услуг отображается в отчете "Agent batches details".

Связь с другими подсистемами

Подсистема "Services" использует данные подсистемы "Shipment" и подсистемы "Customs" для определения параметров аллокации стоимости оказанных услуг на товары.

Данные об увеличении стоимости товаров могут быть использованы в подсистемах фискального учета, таких как: Fixed assets, Inventory и др.
