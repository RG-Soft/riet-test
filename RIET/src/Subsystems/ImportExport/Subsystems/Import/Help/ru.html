Подсистема "Import"

Назначение

Подсистема "Import" входит в подсистему "Import / export logistics" и предназначена для отражения процессов Green light, а также для отражения и мониторинга этапов консолидации и перевозки импортируемых грузов.

Структура подсистемы

Центральным документом подсистемы является документ "DOC" (консолидированный пакет заявок на перевозку).
В нем перечисляются входящие в него документы "Invoices" и грузовые места "Parcels".
В документах "Invoices" перечисляются товары "Goods", которые потом раскладываются по "Parcels".

Документы "Invoices" могут быть автоматически загружены в систему из pdf-файлов с помощью обработки "Invoice loading".
После этого в систему могут быть подгружены документы "DOC" и привязынные к ним "Parcels" с помощью обработки "DOC loading".

Приоретизация и мониторинг состояния документов "DOC", находящихся в работе, производится на рабочем столе специалиста по импорту.

Статистика по длительности нахождения DOCs на определенных этапах отображается в отчете "GL tracking".

Связь с другими подсистемами

Исходными данные о заказанных товарах предоставляются подсистемой "PO".

После обработки DOCs консолидируются для отправки в страну назначения.
Этапы перевозки и таможенного оформления отражаются в подсистеме "Shipment".

Текущее состояние импортируемых товаров доступно внешним пользователям с помощью подсистемы "Russia import export tracking".
Специалисты по импорту также могут воспользоваться отчетом "PO tracking advanced".
