Подсистема "Local distribution"

Назначение

Подсистема Local distribution решает задачи консолидации импортных и локальных грузовых мест для перевозки по территории России.

Структура подсистемы

Локальные грузовые места загружаются с помощью обработки Local parcels loading.
А исходные данные по импортным грузовым местам предоставляются подсистемой Shipment.
В подсистеме Shipment автоматически определяется некоторые реквизиты грузовых мест, необходимых для локальной транспортировки:

  • Warehouse from - место отправления груза локальной транспортировки определяется из реквизита Actual POA документа Shipment
  • Warehouse to - место назначения локальной транспортировки определяется из реквизита Operating location документа Invoice
  • Urgency - срочность определяется из одноименного реквизита документа DOC

Локальные и импортные грузовые места консолидируются в документы Trips с помощью рабочего стола Local distribution desktop.

После заполнения всех необходимых данных в документе Trip данные выгружаются на склад, а потом в систему Move it.

Связь с другими подсистемами

Исходные данные для подсистемы берутся из нескольких источников:

  1. Данные о импортных грузовых местах берутся из подсистемы Shipment
  2. Данные о локальных грузовых местах, по которым есть PO, берутся из Excel-файлов, предоставляемых сладами, и собственно из PO.
  3. Остальные локальные non-PO грузовые места вводятся вручную.
