Документ "Export shipment"

Документ входит в подсистемы "Export", "Customs documents", "Shipment closing" и "Services".

Документ предназначен для отражения консолидированной поставки заявок на экспорт.
Консолидация заявок на экспорт позволяет оформлять одну декларацию на товары на несколько заявок на экспорт, а также осуществлять перевозку грузов по этим заявкам в одном контейнере.

Документ можно ввести на основании заявки на экспорт, при этом эта заявка добавится в таблицу "Export requests".

Реквизиты шапки

  • No. (строка) - автоматически присваеваемый уникальный номер документа.
  • Consignee (справочник Shippers) - автоматически заполняемый из заявок на экспорт получатель грузов.
  • CCA (справочник Agents) - автоматически заполняемый из заявок на экспорт получатель грузов.
  • Int. freight agent (справочник Freight forwarders) - автоматически заполняемый из заявок на экспорт получатель грузов.
  • POD (справочник Sea and AirPorts) - автоматически заполняемый из заявок на экспорт получатель грузов.
  • International MOT (справочник MOTs) - автоматически заполняемый из заявок на экспорт способ международной транспортировки.
  • POA (справочник HUBs / Countries) - автоматически заполняемый из заявок на экспорт страна назначения грузов.
  • Shipping invoice ready (дата)
  • Consignee GL requested (дата)
  • Consignee GL received (дата)
  • Submitted to customs (дата)
  • Released from customs (дата)
  • International ETD (дата)
  • International ATD (дата)
  • Pre alert sent (дата)
  • International ETA (дата)
  • International ATA (дата)
  • International WB1-4 (строка)
  • Export requests no. (строка) - список, в котором система автоматически перечисляет номера заявок на экспорт.
    Это удобно для поиска нужного документа в списке.
  • Created by (справочник Users) - пользователь, который создал документ.
  • Date (дата и время) - дата создания документа.
  • Modified by (справочник Users) - пользователь, который внес последние интерактивные изменения.
  • Modification date (дата и время) - дата последнего интерактивного изменения документа.

Таблица "Export requests"

В таблице перечисляются заявки на экспорт, которые входят в текущую экспортную поставку.

  • Export request (документ Export request)
    Система автоматически удаляет пустые и дублирующиеся строки.
    Export request не может быть отмененным.
    В Export request должно быть заполнено поле Accepted.
    Следующие реквизиты заявок должны совпадать с одноименными реквизитами шапки: Consignee, CCA, International freight agent, POD, Int. MOT и POA.
    Export request не может входить в другую экспортную поставку.
    Система позволяет добавить в таблицу сразу несколько заявок на экспорт, при этом она накладывает фильтры, описанные выше.

Проведение (для разработчиков)

При наступлении события Submitted to customs для каждого товара добавляется приход по регистру накопления Export items without customs files. Приход означает, что товар нужно привязать к декларации.
