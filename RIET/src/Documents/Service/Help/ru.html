Документ Service

Документ Service входит в подсистему "Services".
Предназначен для отражения услуги, оказанной поставщиком, стоимость которой должна быть отнесена на определенную группу товаров, входящих в imp./exp. shipment, customs receipt order (ТПО) или temporary imp./exp. transaction.

ВНИМАНИЕ! Все суммы указываются в рублях!

Реквизиты шапки документа

  • No. (строка) - автоматически присваиваемый номер документа.
  • Agent no. (строка) - номер документа поставщика.
  • Date (дата)
    Обязательный для заполнения реквизит.
  • For deletion (булево) - признак того, что документ помечен на удаление.
  • Posted (булево) - признак того, что документ проведен.
    Помеченный на удаление документ не может быть проведен.
  • Agent (справочник Agents)
    Обязательный для заполнения реквизит.
  • Document base (документ Import Shipment, Export Shipment, Customs files (light) с типом ТПО, Temporary imp./exp. transactions) - документ, к которому относятся товары.
    Обязательный для заполнения реквизит.
    Должен быть проведен (если это не так - система пытается провести документ).
    В документе Shipment должны быть заполнены реквизиты: Actual POA, In customs, причем дата In customs не должна быть больше даты Сервиса.
    В документе ТПО и Temporary imp./exp. должна быть заполнена дата, причем она не должна быть больше даты Сервиса.

  • Customs file (документ Customs file) - декларация, на товары которой будет распределяться стоимость сервиса.
    Реквизит должен быть заполнен в том случае, если Allocation area = Customs file.
    В остальных случаях система автоматически очищает этот реквизит и не показывает его на форме.
    Агент в Customs file должен совпадать с агентом сервиса.
    Customs file должна быть проведена.
    Customs file должна относиться к поставке, указанной в поле Document base.
    При выборе Customs file система автоматически накладывает необходимые фильтры.

  • Days in customs (число) - количество дней затраченных на оформление декларации.
    Реквизит должен быть заполнен, если Allocation area = Customs file.
    В остальных случаях система не показывает его на форме.
  • Service (справочник Service)
    Обязательный для заполнения реквизит.
    Услуга не должна быть помечена на удаление.
    Для этой услуги должен быть действующий прайс-листа.
    Услуга должна быть разрешена для использвания в данном POA (определяется из документа Shipment).
    В услуге должны быть заполнены Allocation area, Sum calculation method.
    При выборе услуги система автоматически накладывает необходимые фильтры.
  • Description (строка)
  • Qty (число)
    Реквизит должен быть заполнен, если указан метод "Standart tariff".
    В остальных случаях он не показывается на форме.
  • Price (число)
    Реквизит должен быть заполнен, если указан метод "Standart tariff".
    В остальных случаях он не показывается на форме.
  • Base (число)
    Рреквизит должен быть заполнен, если указан метод "Cost-plus".
    В остальных случаях он не показывается на форме.
  • Percent, % (число)
    Рреквизит должен быть заполнен, если указан метод "Cost-plus".
    В остальных случаях он не показывается на форме.
  • Markup (число)
    Рреквизит должен быть заполнен, если указан метод "Cost-plus".
    В остальных случаях он не показывается на форме.
  • Sum (число)
    Обязательный для заполнения реквизит.
  • Discount (число)
  • Grand total (число)
    Обязательный для заполнения реквизит.  

Табличная часть "Items"

В табличной части перечисляются товары, на которые должна быть распределена стоимость сервиса.
Табличная часть становится доступной только в том случае, если Allocation area = Goods.
В остальных случаях система автоматически заполняет эту табличную часть перед записью документа.

  • Item (справочник Items)
    Должна быть добавлена хотя бы одна строка.
    Товары должны относится к документу-основанию, указанному в шапке.
    Не должно быть строк с пустыми строками инвойса (система автоматически удаляет такие строки)
    Строки не должны повторяться (система автоматически удаляет такие строки).
    При выборе система автоматически накладывает необходимые фильтры.
    Справочно отображаются реквизиты: Part no., PO line

Проведение

При проведении, в случае если указана услуга с признаком Duties recharge, система списывает неоплаченную сумму декларации или ТПО.
При этом контролируется, что сумма услуги не превышает неоплаченную сумму.
