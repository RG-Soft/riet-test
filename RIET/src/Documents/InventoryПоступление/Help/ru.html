Документ "Inventory поступление"

Документ "Inventory поступление" предназначен для отражения факта поступления inventory-товаров.

Документ создается автоматически при загрузке Lawson отчета XR220. Однако в этом отчете содержатся только количественные данные. Суммовые данные необходимо заполнить вручную или полуавтоматически из подсистемы "Tengible assets".

Реквизиты документа

У документа есть несколько реквизитов шапки и подчиненный справочник, в котором хранятся строки документа.

Реквизиты шапки

  • Inventory location (справочник Inventory locations) - место поступления товаров
    • Реквизит должен быть заполнен
  • Segment (справочник Segments) - реквизит, определяемый автоматически из проводок DSS
  • Ict doc type (строка) - вид операции - справочный реквизит, подгружаемый из lawson
  • PO number (строка) - справочный реквизит, подгружаемый из lawson
    • Реквизит должен быть заполнен
  • PO (справочник POs)
    • Система автоматически пытается заполнить этот реквизит при изменении PO number, а также при ручном выборе фильрует список PO по PO number
  • Has finished goods (булево) - автоматически устанавливаемый признак того, что в документе содержатся Finished goods
  • Creation date (дата и время) - автоматически заполняемая дата создания документа
  • Last modified (дата и время) - автоматически заполнеяемая дата последнего изменения документа из формы
  • Responsible (справочник Users) - пользователь, внесший последние изменения в документ
  • Comment (строка) - любой произвольный комментарий

Строки документа

Информация о товарах, количестве и сумме хранится в специальном справочнике "Строки inventory-поступлений".
Справочник имеет несколько следующие реквизиты:

  • Good (справочник Inventory goods) - товар
    • Реквизит должен быть заполнен
    • Good не может быть помечен на удаление
    • У Good должен быть заполнен реквизит Standard UOM
  • PO line (справочник PO lines) - строка заявки на закупку
    • Указывается для определения фискальной цены из модуля Tengible assets
    • Над таблицей располагается кнопка "Заполнить пустые PO lines", которая заполняет этот реквизит по следующему алгоритму
      • Проверяется заполнение PO number. Если он не заполен - выдается сообщение об ошибке.
      • От PO number отсекается "-0-SWPS", если такой присутствует
      • Ищется PO по PO number. Если PO не находится - выдается сообщение об ошибке.
      • Если в PO только одна строка - она и подставляется.
      • Среди найденных PO lines ищется та, у которой Supplier P / N совпадает с кодом Good из строки документа. Если таких строк нет или найдено несколько строк - выдается сообщение об ошибке.
    • Автоматическое определение PO line возможно не всегда, поэтому система позволяет выбрать PO line вручную, при этом при открытии формы выбора PO line система добавляет фильтры по номеру PO и по Supplier P / N.
  • Finished good (булево) - признак того, что для данного товара устанавливается ручной порядок списания
  • UOM (справочник UOMs) - единица измерения
    • Реквизит должен быть заполнен
    • UOM не может быть помечен на удаление
    • UOM standard UOM должен соответствовать Good standard UOM
  • Qty (число) - количество
    • Реквизит должен быть заполнен
  • Fiscal price (число) - фискальная цена
  • Fiscal sum (число) - фискальная сумма
    • Реквизит должен быть заполнен
  • Management price (число) - управленческая цена - указывается справочно
  • Management sum (число) - управленческая сумма - указывается справочно

Проведение документа

При проведении документ "Поступление" делает приходные записи в регистр "Inventory партии":

  • Inventory location - из шапки документа
  • Good - из строки документа
  • PO number - из шапки документа
  • Batch - сам документ "Поступление" 
  • Quantity - из строки документа
  • Sum - фискальная сумма из строки документа
