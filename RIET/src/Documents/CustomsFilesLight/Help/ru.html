Документ "Customs files (light)"

Документ "Customs files (light)" входит в подсистему "Customs documents".
Предназначен для ручного отражения таможенных документов. Для каждого вида документа создан отдельный тип транзакции:

  • Customs file (декларация) - таможенная декларация.
  • Customs receipt order (ТПО) - таможенный приходный ордер.
  • Customs bond (расписка) - таможенная расписка за депозит. 
  • Customs bond closing (решение по расписке) - решение по таможенной расписке.

Отражение Customs files (light) позволяет в дальнейшем осуществить:

  • аллокацию таможенных платежей (в подсистеме "Payments")
  • учесть пошлины, сборы и НДС для выгрузки в Lawson (в подсистеме "Batches of customs files")
  • отразить стоимость услуг брокера для выгрузки в Lawson (в подсистеме "Services")
  • отразить КТС для выгрузки в Lawson
  • отслеживать решение по ТР, получение суммы к возврату по депозиту (если решение принято в пользу компании)
  • отразить пени по решению таможенной расписки (тип транзакции - ТПО)
  • отразить заявление на возврат денег на банковский счет с платежки (тип транзакции - ТПО)

Кроме того, документ позволяет выполнить закрытие поставки для ТПО и декларации.

Отражение Customs bonds (таможенных расписок)

  1. Отражение декларации
    После загрузки документа Customs file необходимо проверить, что суммы указаны без корректировок.
    Перед занесением в систему Customs bond, нужно проверить, что заведен документ     Invoice lines matching.
  2. Отражение Customs bond
    Создаем Customs file light с типом транзакции Customs bond на основании загруженной Customs file или из списка документов Customs files (light).

    В таблице Items указываем товары, по которым есть корректировка.
    В таблице Payments указываем общую сумму увеличения платежей одной строкой на тип платежа 9090.
    На вкладке Customs payments allocations создаем аллокацию с Customs payment.
    При формировании Batches of customs files Customs bond попадает на вкладку Customs bonds и выгружается в Lawson:
    Account for Deposit "141102";
    Sub Account for Deposit "141102-1";
    AU for Deposit "2812019", Activity пустой.

  3. Отражение решения таможни - Customs bond closing
    Создаем Customs file light с типом транзакции Customs bond closing (на вкладке Customs bond closing в форме Customs bond или из списка документов Customs files (light)).
    В таблице Items указываем товары, по которым принято решение (система дает выбрать только те товары, по которым еще нет решения).

    Заполнение сумм платежей зависит от принятого решения:

    - если таможня решила, что корректировка будет в полном объеме, необходимо заполнить таблицу Payments на суммы корректировок с разбивкой по типам платежей.  
    - если таможня решила, что корректировки не будет, необходимо заполнить поле amount to refund и выбрать способ Refund to.
    - если таможня решила, что корректировка будет, но в меньшем объеме, необходимо заполнить таблицу Payments на суммы корректировок, заполнить частичную сумму к возврату в поле amount to refund и выбрать способ Refund to. 

    При проведении документа система автоматически распределит окончательные платежи на выбранные товары (если решени принято полностью или частично в пользу таможни).
    При формировании Batches of customs files Customs bond closing попадает на вкладку Customs bonds closing и выгружается в Lawson.
    Для товаров выгрузка в батч аналогична выгрузке затрат на товары по декларациям, для Amount to refund сумма будет с минусом на те же счета и косты, как для депозита.
  4. Отражение возврата денег от таможни
    Создаем Customs deposit refund to bank account на вкладке Deposit refund to bank account в форме Customs bond closing или из списка документов Customs deposit refund to bank account.
    Указываем Customs bond и сумму, которую вернула таможня.
    Проведение документа оповещает систему о факте возврата суммы депозита таможней на банковский счет.

Структура документа

Реквизиты шапки документа

  • No. (строка)
    Заполняется автоматически при записи документа как [номер таможенного поста]-[дата документа в формате ЧЧММГГ]-[seq no.]
    Контролируется уникальность.
  • Type of transaction (перечисление Types of Customs file (light) transaction)
    Обязательный для заполнения реквизит.
  • Customs post (справочник Customs posts)
    Обязательный для заполнения реквизит.
    Таможенный пост не может быть помечен на удаление.
  • Date (дата)
    Обязательный для заполнения реквизит.
    Дата не может быть меньше 01.01.2011 или больше текущей даты.
  • For deletion (булево) - признак того, что данный документ помечен на удаление.
  • Posted (булево) - признак того, что данный документ проведен.
  • Seq. no. (строка)
    Обязательный для заполнения реквизит.
  • I / E (перечисление Import / Export)
    Обязательный для заполнения реквизит.
    Если заполнен Shipment, то I / E должен соответствовать одноименному реквизиту в Shipment.
  • CCA (справочник Agents) - customs clearance agent.
    Обязательный для заполнения реквизит.
    Таможенный брокер не может быть помечен на удаление.
    Если заполнен Shipment, то I / E должен соответствовать одноименному реквизиту в Shipment.
  • Legal entity (справочник Parent company) - юр. лицо - покупатель товаров.
    Обязательный для заполнения реквизит.
    Покупатель не может быть помечен на удаление.
    Если заполнен Shipment, то покупатель должен входить в список покупателей, входящих в Shipment.
  • Invoice currency (справочник Валюты) - валюта инвойса, в соответствии с которым оформляются товары.
    Обязательный для заполнения реквизит.
    Валюта не может быть помечена на удаление.
    Если заполнен Shipment, то валюта должна входить в список валют инвойсов, входящих в Shipment.
  • Invoice currency rate (число)
    Обязательный для заполнения реквизит.
    Курс не может быть отрицательным.
  • Customs value (число) - таможенная стоимость товаров.
    Обязательный для заполнения реквизит.
    Валютная стоимость товаров * курс не может быть больше таможенной стоимости.
  • Release date (дата)
    Обязательный для заполнения реквизит.
    Дата выпуска должна быть больше или равна дате документа, но не больше текущей даты.
    Если заполнен Shipment, а в нем заполнен реквизит Cleared, то дата выпуска не может быть больше даты Cleared.
  • Regime (справочник Customs regimes)
  • Permanent/Temporary -определяет относится декларация к временному ввозу или нет.
  • Expiry date (дата) - дата истечения срока временного импорта / экспорта.
  • Shipment (документ Shipment) - поставка, обязательна для заполнения для всех типов транзакций, кроме ТПО, 
    для ТПО заполяется, когда поставка закрывается документом ТПО, а не документами "Customs file" и "Invoice lines matching"
    Поставка должна быть проведена, если она не проведена - система пытается автоматически провести ее.
  • Customs file (документы Customs file, Customs files (light)) 
    Обязательный для заполнения реквизит для Customs bond и Customs bond closing.
    Для Customs bond происходит отбор "Customs file" с фильтром по поставке.
    Для Customs bond closing происходит отбор Customs files (light), по которым еще не было принято решение.
  • amount to refund - (число) - сумма, которую обязана вернуть таможня после принятия решения по ТР,
    сумма не может быть больше суммы депозита.
  • refund to - способ возврата депозита по таможанной расписке.
  • Paid by CCA (булево) - признак того, что пошлины и сборы оплачиваются непосредственно таможенным брокером, а потом перевыставляются компании Шлюмберже
  • Creation date (дата и время) - дата создания документа, заполняется автоматически при первой записи документа в базу
  • Last modified (дата и время) - дата последнего интерактивного изменения документа, заполняется автоматически при записи документа из формы
  • Responsible (справочник Users) - пользователь, который последним внес изменения в документ, заполняется автоматически при записи документа из формы

Табличная часть "Items"

В табличной части перечисляются товары, которые относятся к данной декларации.

  • Item (справочник Items)
    Если товар импортный, то соответсвующий инвойс должен быть отмечен как Final.
  • Customs file HTC (строка) - обязателен для Customs file light.
  • Customs file net weight (число) - обязателен для Customs file light.

Табличная часть "AUs"

В табличной части перечисляются AU, на которые относится стоимость товаров, оформленных по ТПО.

  • AU (справочник AUs)
    Обязательный для заполнения реквизит.
    Кост-центр не может быть помечен на удаление.
    Если заполнен реквизит PO line, то AU в PO line должен соответствовать AU в строке табличной части.
  • Description (строка)
    Обязательный для заполнения реквизит.
  • Invoice total value (число) - стоимость товаров по инвойсу.
    Обязательный для заполнения реквизит.
  • PO line (справочник PO lines)
    PO line не может быть помечена на удаление.
    При изменении PO line происходит заполнение AU и Description.

Табличная часть "Payments"

Обязательна для заполнения для Customs bond, Customs file light (кроме случаев, когда в PSA указан признак Zero customs charges)

  • Payment kind (строка из 2х символов)
    Обязательный для заполнения реквизит.
  • Sum (число)
    Обязательный для заполнения реквизит.
  • Paid by CCA

Вкладка "Customs payments allocations"

На вкладке "Customs payments allocations" отображается список всех аллокаций таможенных платежей, привязанных к данному Customs file light.
Состав колонок и поведение вкладки аналогично одноименной вкладке в документе Customs file.

Вкладка "Services"

На вкладке "Services" отображается список всех сервисов, привязанных к данному ТПО.
Состав колонок и поведение вкладки аналогично одноименной вкладке в документе Shipment.

Проведение

При проведении происходит проверка заполнения реквизитов, а также выполняются следующие операции:

Отражение неоплаченных ТПО

Для каждой строки из табличной части "Payments" происходит добавление записи в регистр "Неоплаченные ТПО":

  • ТПО - ссылка на текущий документ
  • Payment kind - одноименный реквизит из строки табличной части
  • Sum - одноименный реквизит из строки табличной части

Разнесение платежей по кост-центрам для дальнейшей выгрузки в Lawson

Для ТПО суммы из табличной части "Payments" разносятся на строки табличной части "Goods" пропорционально стоимости товаров по инвойсу.
Далее эти суммы отражаются в регистре "Стоимость товаров по ТПО".
Для ТР суммы из табличной части "Payments" разносятся на строки табличной части "Items" пропорционально стоимости товаров по инвойсу.
Далее эти суммы отражаются в регистре "Invoice lines costs".

  • Document-base - ссылка на текущий документ
  • AU - одноименный реквизит из строки табличной части "Goods"
  • Account - ???
  • Activity - Default activity из AU
  • Duty - расчитанная сумма сбора
  • Fee - расчитанная сумма пошлины
  • VAT - расчитанная сумма НДС

Проведение (для разработчиков)

Для каждого товара добавляется расход по регистру накопления Import items without customs files, если это импортная поставка,
или по регистру накопления Export items without customs files, если это экспортная поставка. 
Расход по данным регистрам позволяет отследить, что товар привязан к декларации.
