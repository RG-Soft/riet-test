Документ "Customs payment"

Документ "Customs payment" входит в подсистему "Payments".
Предназначен для отражения платежей, перечисляемых таможне для оплаты пошлин и сборов.

Документы могут быть автоматически загружены из текстового файла, который формируется клиентом City банка, с помощью обработки "Customs payments loading".

При проведении документа происходит отражение несаллокированной с CCD или Customs files (light) суммы.

В форме списка документов красным цветом отображаются несаллокированные платежи, а зеленым - саллокированные.

Для аллокации платежа необходимо создать из текущего документа документ "Customs payment allocation". После создания "Customs payment allocation" реквизиты текущего документа нельзя изменять.

Структура документа

  • No. (строка)
    Обязательный для заполнения реквизит
    Не должно быть другого документа Customs payment с таким же номером.
  • Date (дата)
    Обязательный для заполнения реквизит.
  • For deletion (булево) - признак того, что документ помечен на удаление.
  • Posted (булево) - признак того, что документ проведен.
  • Type of payment (перечисление Types of customs payments) 
    Обязательный для заполнения реквизит.
  • Customs (справочник Customs) 
    Обязательный для заполнения реквизит.
    Выбранный элемент не может быть помечен на удаление. 
  • Bank account (справочник Bank accounts for customs payments)
    Обязательный для заполнения реквизит.
    Выбранный элемент не может быть помечен на удаление.
    Вид платежей банковского счета должен совпадать с видом платежа, указанном в шапке документа.
  • CCD ref. (строка) - часть номера ГТД.
    Может быть автоматически определен из описания платежа.
  • Customs card (справочник Customs cards).
  • Check no. (строка) - номер чека
    Может быть автоматически определен из описания платежа.
  • Payment kind (строка)
    Может быть автоматически определен из описания платежа.
  • Sum (число) 
    Обязательный для заполнения реквизит. 
    В форме документа справочно отображается текущий несаллокированный остаток.
  • Description (строка) - описание платежа.
  • Creation Date (дата) - дата создания документа
  • Last modified (дата и время) - заполняется автоматически датой последнего интерактивного изменения документа.
  • Responsible (справочник Users) - заполняется автоматически пользователем, который внес последние изменения в документ.
