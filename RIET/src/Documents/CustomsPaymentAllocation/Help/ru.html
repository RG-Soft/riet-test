Документ "Customs payment allocation"

Документ "Customs payment allocation" входит в подсистему "Payments".
Документ предназначен для аллокации суммы платежа с одной стороны и Customs file или Customs file (light) - с другой.

Документ может быть создан как из документа "Customs payment", так и из Customs file, причем в последнем случае возможен автоматический подбор платежей.

При проведении документа происходит списание несаллокированной суммы платежа и неоплаченный суммы Customs file по данному коду платежа.

Реквизиты шапки

  • No. (строка) - автоматически присваеваемый номер документа.
  • Date (дата)
    Обязательный для заполнения реквизит.
  • For deletion (булево) - признак того, что документ помечен на удаление.
  • Posted (булево) - признак того, что документ проведен.
  • Customs Payment (документ Customs payment)
    Реквизит должен быть заполнен
    Платеж должен иметь несаллокированный остаток
    Справочно отображается текущая несаллокированная сумма платежа с учетом добавленных в документ ГТД
  • Payment kind (строка)
    Обязательный для заполнения реквизит. 
  • Customs document (документ CCD или ТПО)
    Реквизит должен быть заполнен.
    Customs, Sold-to и Payment kind должни совпадать с одноименными реквизитами в платеже.
    По документу должен быть неоплаченный остаток.
    При выборе документа в форме подбора справочно отображаются текущие неоплаченные остатки по документам.
  • Sum (число)
    Обязательный для заполнения реквизит.
    Сумма не должна превышать неоплаченный остаток по декларации.
    При выборе customs document сумма заполняется автоматически по следующему алгоритму:
    • если несаллокированная сумма больше суммы ГТД - подставляется вся сумма ГТД
    • если несалокированная сумма меньше суммы ГТД - подставляется несаллокированная сумма
  • Comment (строка)
  • Creation Date (дата) - автоматически заполняемая дата создания документа.
  • Last modified (дата и время) - заполняется автоматически датой последнего интерактивного изменения документа.
  • Responsible (справочник Users) - заполняется автоматически пользователем, который внес последние изменения в документ.
