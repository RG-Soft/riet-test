Документ 'Temporary imp. / exp. transaction'

Документ предназначен для изменения состояния товаров, находящихся во временном импорте или экспорте.

Документ входит в подсистему Temporary import export.

Документ позволяет отразить следующие операции:

  • temporary import - ручная постановка товаров на временный импорт
  • temporary export - ручная постановка товаров на временный экспорт
  • prolongation - продление срока нахождения товаров во временном импорте или экспорте
  • responsible change - изменение ответственного за товары во временном импорте или экспорте
  • reexport - экспорт товаров, находящихся во временном импорте
  • reimport - импорт товаров, находящихся во временном экспорте
  • permanent import - перевод товаров из временного импорта в постоянный
  • permanent export - перевод товаров из временного экспорт в постоянный

Реквизиты шапки

  • No. (текст) - генерируемый системой уникальный номер документа.
  • Transaction date (дата) - дата транзакции. Например для temporary import - это дата импортной декларации, для reexport - дата экспортной декларации и т. д.
    Обязательный для заполнения реквизит.
    Система контролирует хронологическую последовательность операций. Например, дата prolongation не может быть раньше даты принятия во временный импорт.
  • Posted (булево) - признак того, что документ проведен (непроведенный документ - это просто черновик).
  • For deletion (булево) - признак того, что документ помечен на удаление.
  • Process level (справочник Process levels)
    Обязательный для заполнения реквизит.
    Автоматически заполняется тем process level, который указан для текущего пользователя.
  • Type of transaction (перечисление Types of temporary imp. / exp. transaction) - вид операции, перечислены выше.
    Обязательный для заполнения реквизит.
  • Customs file (документ Customs file или Customs file (light)) - ссылка на декларацию, по которой товары были приняты во временный импорт или экспорт.
    Обязательный для заполнения на всех операциях кроме temporary import, temporary export и responsible change (для этих операций очищается и становится невидимым).
    Customs file должен быть проведен.
    Дата выпуска Customs file должны быть раньше даты транзакции.
    Для импортных операций должны использоваться импортные Customs files, а для экспортных - экспортные.
    Customs file должен иметь один из временных режимов.
  • Customs file no. (текст) - номер декларации, по которой товары были приняты во временный импорт или экспорт.
    Этот реквизит необходим потому, что для ручного принятия во временный импорт или экспорт в системе может не быть заведены декларации,
    в этом случае заполняется только поле Customs file no., а поле Customs file остается пустым.
    Во всех остальных случаях это поле заполняется автоматически номером из Customs file.
  • Expiry date (дата) - дата истечения срока временного импорта / экспорта.
    Обязательная для заполнения на операциях temporary import, temporary export и prolongation (в остальных случаях очищается и становится невидимым).
  • New responsible (справочник Users) - ответственный за товары во временном импорте или экспорте.
    Обязательный для заполнения на операциях temporary import, temporary export и responsible change (в остальных случаях очищается и становится невидимым).
  • CCA job ref. (текст)
    Обязательный для заполнения на операциях temporary import и temporary export (в остальных случаях очищается и становится невидимым).
  • Shipper name (текст)
    Обязательный для заполнения на операциях temporary import и temporary export (в остальных случаях очищается и становится невидимым).
  • Customs regime (справочник Customs regimes)
    Обязательный для заполнения на операциях temporary import и temporary export (в остальных случаях очищается и становится невидимым).
    Доступны только режимы временного импорта / экспорта.
    Для операции temporary import доступны только импортные режимы, а для temporary export - только экспортные.
  • New customs file no. (текст) - номер новой декларации.
    Обязательный для заполнения на операциях reexport, reimport, permanent import, permanent export.
  • Comments (текст) - любой произвольный комментарий пользователя.
  • Created by (справочник Users) - автоматически заполняется пользователем, который создал документ.
  • Creation date (дата и время) - автоматически заполняется датой создания документа.
  • Modified by (справочник Users) - автоматически заполняется пользователем, который внес последние изменения в документ.
  • Modification date (дата и время) - автоматически заполняется датой последнего изменения документа.

Для permanent import и permanent export справочно отображаются ссылки Permanent customs file и Closing document (или кнопка Create items matching, если Invoice lines matching еще не создан).

Табличная часть 'Items'

В табличной части 'Items' перечисляются товары, к которым относится данная транзакция.
Для операций temporary import и temporary export над табличной частью отображаются кнопки, которые позволяют быстро создать новый товары или копировать выделенный.
В табличной части должна быть хотя бы одна строка.

  • Item (справочник Items) - товар.
    Обязательный для заполнения реквизит (строки с пустым реквизитом автоматически удаляются).
    Допускаются только товары с галочкой Final (для товаров в импортных инвойсах она ставится автоматически).
    Допускаются только товары, помеченные как Temporary.
    Для операций temporary import и temporary export (ручное принятие во временный импорт / export) разрешаются только вручную созданные товары (они не должны быть в инвойсах или заявках на экспорт).
    Для операций prolongation, reexport, reimport, temporary export, temporary import проверяется, что товары привязаны именно к той декларации, которая указана в шапке.
    Для операций temporary import и temporary export проверяется заполнение количество в самом товаре.
    Для операций temporary import и temporary export проверяется, что эти товары еще не приняты на временный контроль.
  • Qty to transfer (число) - количество товара, выводимое из под временного контроля.
    Обязательный для заполнения на операциях reexport, reimport, permanent import, permanent export.
    Количество товара, выводимое из под временного контроля, не должно привышать изначальное количество товара, указанное в самом товаре.

Вкладка "Services"

На вкладке "Services" отображается список всех сервисов, привязанных к данной транзакции (для permanent import и permanent export).

Контроль изменения

Запрещается изменять документ, если для указанных в нем товаров, уже есть следующие транзакции.

Движения по регистрам (для разработчиков)

  • Qty of items in temporary imp. / exp.
    Операции temporary import и temporary export делают приход товаров на количество, указанное в самих товарах.
    Операции reexport, reimport, permanent import, permanent export делают расход товаров на количество, указанное в табличной части.
  • Customs fils of items
    Операции temporary import и temporary export привязывают перечисленные в табличной части товары к номеру декларации, указанному в Customs file no.
  • Expiry dates of items in temporary imp. / exp.
    Операции temporary import, temporary export и prolongation отражают актуальную expiry dates для товаров, указанных в табличной части.
  • Responsibles for items in temporary imp. / exp.
    Операции temporary import, temporary export и responsible change отражают актуального ответственного за товары, указанные в табличной части.
  • Additional data of items in temporary imp. / exp.
    Операции temporary import и temporary export отражают дополнительную информацию, связанную с товарами, указанными в табличной части.
