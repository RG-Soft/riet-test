Документ "Распределение импорта по закрытию поставки"

Назначение

Документ предназначен для сопоставления данных DSS с данными модуля Customs.

"Распределение импорта по закрытию поставки" фактически представляет собой список Invoice lines, отобранных по конкретному документу "Shipment closing", который нужно сопоставить со списком проводок DSS.

Способы формирования документа

Документ может быть сформирован на основании документа "Shipment closing" или из списка документов.

Структура документа

Реквизиты шапки документа:

- Shipment closing (документ)
- Invoice total sum (общая сумма invoice lines из регистра Invoice lines costs)
- DSS total tran amount (общая сумма transaction amount из проводок DSS)
- Delta (разница между Invoice total sum и DSS total tran amount)
- Список CCD (список CCD из документа Shipment closing)

Табличные части

  • Invoices

содержит список номеров Инвойсов из документа "Shipment closing" (указанного в шапке) и количество строк инвойса, которые осталось сопоставить с проводками DSS в табличной части Tangible assets (Unmatched).

  • Tangible assets

содержит список строк выделенного инвойса, и две колонки для заполнения:

- проводка DSS, которую можно перетащить из таб. части Expenses

- номенклатура для строк инвойса с типом "FA"

По кнопке Match DSS with Invoice lines автоматически производится попытка сопоставить по PO line и по Item code проводки в таб. части Expenses и строки инвойса в таб. части Tangible assets

  • Expenses

содержит список всех проводок DSS, загруженных автоматически (по кнопке Fill) или подобранных пользователем, но еще не сопоставленных со строками инвойса. По кнопке Match by current CCD в форме подбора будет установлен фильтр по Vendor (Контрагент Lawson в Customs post) и DateLawson (Дата ГТД)

В табличную часть подбираются, но не отображаются для сопоставления проводки, у которых в Description указано INSURANCE, HANDLING, FREIGHT.

Проводки можно отфильтровать по выделенному инвойсу, если значение AP invoice соответствует номеру.

Проверки, выполняемые при записи/проведении

- заполнен реквизит Shipment closing

- при проведении не должно быть несопоставленных строк в табличной части Tangible assets

- Delta должна быть равна 0 (т.е. подобраны все проводки DSS)

Проведение

Документ делает расходные движения по регистру "Материальные активы" по всем проводкам DSS, подобранным в документе, выполняется проверка регистра на наличие остатков по проводкам DSS в табличной части.

Документ делает приходные движения по регистру "Оборудование DSS" по проводкам, сопоставленным с Invoice lines с типом "FA".

Документ делает приходные движения по регистру "Inventory tangible assets costs" по проводкам, сопоставленным с Invoice lines с типом "Inventory".
