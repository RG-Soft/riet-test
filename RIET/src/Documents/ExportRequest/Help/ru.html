Документ "Export request"

Документ "Export request" входит в подсистему "Export".

Документ предназначен для отражения заявок на экспорт.
В нем также отражаются все этапы экспорта товаров.

Документ может быть создан внешним пользователем или специалистом по экспорту.
При этом внешний пользователь может посмотреть информацию только по своим заявкам на экспорт.
Проверка полей, которые необходимо заполнить внешнему пользователю, происходит в момент заполнения даты Request date.
При этом внешний пользователь видит только те поля, которые ему необходимо заполнить до заполнения Requeste date (и те, которые явно указаны в детальном описании полей ниже).

После того как специалист по экспорту принимает заявку в работу (заполняется Accept date) - она закрывается для редактирования для внешних пользователей.
Одновременно происходит проверка заполнения некоторых оставшихся полей.

К документу можно прикреплять произвольные файлы.

Ведется история изменений реквизитов документа (логи).

Защита от изменения

ОПИСАТЬ

Реквизиты шапки

  • No. (строка) - номер заявки на экспорт.
    Недоступен для редактирования, но виден внешнему пользователю.
    Заполняется автоматически. Например, E12-0201. Здесь 12 - последние 2 цифры текущего года, 02 - номер месяца, 01 - порядковый номер в текущем месяце.
  • Export purpose (справочник Export purposes) - цель экспорта.
    Обязательный для заполнения, если заполнена дата Submitted.
    Не может быть помечен на удаление.
    При интерактивном выборе система скрывает неподходящие значения.
  • Export purpose description (строка) - описание цели экспорта.
    Отображается и проверяется на заполнение только тогда, когда в поле Export purpose указано значение Other.
  • Export mode (перечисление Export modes) - temporary или permanent.
    Обязательный для заполнения специалистом по экспорту, если заполнена Accepted by export specialist.
  • RAN number (строка) - Repair authentication number.
    Может быть заполнен внешним пользователем.
  • Urgency (перечисление Urgencies) - важность: Standard, Urgent, Emergency.
    Обязательный для заполнения, если заполнена дата Submitted.
  • Urgency comment (строка) - комментарий по срочности.
    Обязательный для заполнения и виден только для Urgent и Emergency.
  • Required delivery date (дата) - требуемая дата доставки груза.
    Обязательный для заполнения и виден только для Emergency.
    Не может быть раньше даты Submitted + 3 дня.
    Не может быть раньше Ready to ship + 1 день.
  • Emergency approved by (строка) - alias сотрудника, который подтвердил, что груз действительно имеет приоритет срока над стоимостью.
    Обязательный для заполнения и виден только для Emergency.
    Должен быть не короче 3х символов.
  • Ready to ship date (дата) - дата, к которой груз будет готов к вывозу со склада отправления.
    Обязательный для заполнения, если заполнена дата Submitted.
    Не может быть раньше даты создания.
  • Pick up from (строка) - место отправления товара.
    Обязательный для заполнения, если заполнена дата Submitted.
  • Pick up from address, contact, phone, e-mail (строковые реквизиты)
    Обязательны для заполнения, если заполнена дата Submitted.
  • Company (справочник Sold-to) - компания-продавец товара.
    Обязательный для заполнения, если заполнена дата Submitted.
    Не может быть помечен на удаление.
    При интерактивном выборе система скрывает неподходящие значения.
  • Shipper (справочник Consign-to)
    Обязательный для заполнения, если заполнена дата Submitted.
    Shipper company должна соответствовать продавцу товара.
    Не может быть помечен на удаление.
    При интерактивном выборе система скрывает неподходящие значения.
  • Shipper address, contact, phone, e-mail (строковые реквизиты)
    Обязательны для заполнения, если заполнена дата Submitted.
  • Deliver-to company, address, contact, phone, e-mail (строковые реквизиты)
    Обязательны для заполнения, если заполнена дата Submitted.
  • Consignee (cправочник Shippers)
    Обязательный для заполнения, если заполнена дата Accepted by specialist.
    Не может быть помечен на удаление.
    При интерактивном выборе система скрывает неподходящие значения.
  • Consignee company, address, contact, phone, e-mail (строковые реквизиты)
    Обязательны для заполнения, если заполнена дата Submitted.
  • Segment (справочник Segments) 
    Обязательный для заполнения, если заполнена дата Submitted.
    Segment AU и БОРГа (если заполнен) должен совпадать с Segment.
  • BORG (справочник BORGs)
    Обязательный для заполнения, если заполнена дата Submitted.
    Не может быть помечен на удаление.
    Компания БОРГа должна совпадать с Company.
    При интерактивном выборе система скрывает неподходящие значения.
  • AU (справочник AUs) - accounting unit - кост-центр
    Обязательный для заполнения, если заполнена дата Submitted.
    Не может быть помечен на удаление.
    При интерактивном выборе система скрывает неподоходящие значения.
  • Activity (строка)
    Обязательный для заполнения, если заполнена дата Submitted.
  • Submitted (дата документа) - дата заявки на экспорт.
    Внешний пользователь не может изменить эту дату вручную, а только нажать на кнопку Sent to export specialist, которая заполняет эту дату и записывает заявку на экспорт.
    Не может быть раньше даты создания заявки.
    Не может быть позже текущей даты.
  • Submitter (справочник Users) - составитель заявки на экспорт.
    Заполняется автоматически текущим пользователем.
    Не может быть изменен внешним пользователем, но отображается на форме.
    Обязательный для заполнения, если заполнена дата Submitted.
  • Export specialist (справочник Users) - специалист по экспорту, который занимается экспортом данной заявки.
    Заполняется автоматически при первом заполнении даты Accepted by specialist, но может быть в дальнейшем изменено вручную.
  • Accepted by specialist (дата) - дата принятия заявки в работу специалистом по экспорту.
    Не может быть раньше даты создания заявки.
    Не может быть раньше даты Sent to export specialist, если она заполнена.
    Не может быть позже текущей даты.
  • Pending, days (число) - число дней, прошедшее с даты Accepted by specialist.
    Обновляется автоматически до тех пор, пока не будет заполнена дата ATD.
  • Current comments (строка) - актуальные комментарии.
  • Gross weight, kg. (число) - вес брутто всех грузовых мест.
    Рассчитывается автоматически.
  • Pieces qty (число) - количество грузовых мест.
    Рассчитывается автоматически.
  • Cubic meters (число) - объем всех грузовых мест.
    Рассчитывается автоматически.

Таблица "Items"

В таблице "Items" перечисляются все товары, которые должны быть экспортированы.

Технически - это не таблица, а специальный справочник Goods.
Реквизиты этого справочника описаны в соответствующем разделе справки.

Если заполнена Submitted date - то в таблице должен быть указан хотя бы один товар.

Таблица "Parcels"

В таблице "Parcels" перечисляются все грузовые места (Parcels), которые относятся к этой заявке на экспорт.

Технически - это не таблица, а специальный справочник "Parcels".
Реквизиты этого справочника описаны в соответствующем разделе справки.

Если заполнена Submitted - то в таблице должено быть указано хотя бы одно грузовое место.

Local delivery

  • Local freight requested (дата) - дата запроса цены у агента.
    Обязательный для заполнения, если заполнена дата Local freight received.
    Не может быть раньше даты Submitted.
    Не может быть позже текущей даты.
  • Local freight received (дата) - дата получения Green light от брокер.
    Обязательный для заполнения, если заполнена дата Local freight approved.
    Не может быть раньше даты Local delivery requested.
    Не может быть позже текущей даты.
  • Local freight approved (дата) - дата подтверждения стоимости локальной транспортировки со стороны компании Шлюмберже.
    Не может быть раньше даты Local delivery received.
    Не может быть позже текущей даты.
  • Local freight agent (справочник Agents) - агент, который будет проводить локальную транспортировку товара.
    Обязательный для заполнения, если заполнена дата Local freight approved.
    Не может быть помечен на удаление.
  • Pick up warehouse (справочник Warehouses) - склад отправления груза.
    Не может быть изменен внешним пользователем, поэтому заполняется специалистом по экспорту перед принятием заявки в работу.
    Обязательный для заполнения и не может быть помечен на удаление, если заполнена дата Accepted by export specialist.
  • Local MOT (справочник MOTs) - способ локальной транспортировки.
    Обязательный для заполнения, если заполнена дата Local freight approved.
    Не может быть помечен на удаление.
  • Local WB (строка) - накладная на локальную транспортировку.
    Обязательный для заполнения, если заполнена Pick-up date.
  • Local ETD (дата) - планируемая дата отправки с локального склада-отправителя.
    Не может быть раньше даты Local freight received.
  • Local ATD (дата) - фактическая дата отправки с локального склада-отправителя.
    Обязательный для заполнения, если заполнена дата Local ATA.
    Не может быть раньше даты Local freight received.
    Не может быть позже текущей даты.
  • Local ETA (дата) - планируемая дата прибытия на промежуточный склад.
    Не может быть раньше Local freight received, Local ETD или Local ATD.
  • Local ATA (дата) - фактическая дата прибытия на промежуточный склад.
    Не может быть раньше Local ATD.
    Не может быть позже текущей даты.

Customs

  • CCA GL requested (дата) - дата запроса Green light у брокера.
    Обязательный для заполнения, если заполнена дата CCA received.
    Не может быть раньше даты Submitted.
    Не может быть позже текущей даты.
  • CCA GL received (дата) - дата получения Green light от брокер.
    Обязательный для заполнения, если заполнена дата CCA approved.
    Не может быть раньше даты Submitted.
    Не может быть позже текущей даты.
  • Source docs available (перечисление Yes / no) - наличие или отсутствие импортных документов.
    Должен быть заполнен, если заполнена дата CCA received.
  • CCA (справочник Agents) - брокер, который будет проводить таможенное оформление товара.
    Обязательный для заполнения, если заполнена дата CCA received.
    Не может быть помечен на удаление.
  • Permits required (перечисление Yes / no) - требуется разрешающие документы.
    Обязательный для заполнения, если заполнены даты Permits requested, Permits obtainted или Submitted to customs.
  • Permits requested (дата) - разрешающие документы запрошены.
    Обязательный для заполнения, если заполнена дата Submitted to customs и Permits required.
    Не может быть раньше даты Submitted.
    Не может быть позже текущей даты.
  • Permits obtained (дата) - разрешающие документы получены.
    Обязательный для заполнения, если заполнена дата Submitted to customs и Permits required.
    Не может быть раньше даты Permits requested.
    Не может быть позже текущей даты.
  • Shipping invoice ready (дата) - дата получения shipping invoice.
    Не может быть раньше даты CCA received.
  • Consignee GL requested (дата) - дата запроса Green light у получателя товара
    Обязательный для заполнения, если заполнена дата Consignee GL received.
    Не может быть раньше даты Submitted.
    Не может быть позже даты Consignee GL received. 
  • Consignee GL received (дата) - дата получения Green light получателем товара
    Не может быть раньше даты GL requested from Consignee.
  • Fumigation required (перечисление Yes / no) - требуется фумигация.
    Обязательный для заполнения, если заполнена дата Fumigation done или Submitted to customs.
  • Fumigation certificate required (перечисление Yes / no) - требуется сертификат о фумигации.
    Обязательный для заполнения, если заполнена дата Fumigation done или Submitted to customs.
  • Fumigation done (дата) - фумигация выполнена.
    Обязательный для заполнения, если заполнена дата Submitted to customs и Fumigation required или Fumigation certificate required.
  • Submitted to customs (дата) - дата подачи документов для таможенного оформления.
    Обязательный для заполнения, если заполнена дата Released.
    Не может быть раньше даты Consignee GL received.
    Не может быть позже текущей даты.
  • Released from customs (дата) - дата выпуска таможенной декларации.
    Не может быть раньше даты submitted to customs.
    Не может быть позже текущей даты.
  • CCD number (строка) - номер ГТД.
    Обязательный для заполнения, если заполнена дата Released from customs.

International delivery

  • International freight requested (дата) - дата запроса цены у агента.
    При добавлении новой строки подставляется автоматически текущая дата, но может быть изменена.
    Обязательный для заполнения, если заполнена дата International freight received.
    Не может быть раньше даты Submitted.
    Не может быть позже текущей даты.
  • International freight received (дата) - дата получения цены от агента.
    Обязательный для заполнения, если заполнена дата International freight approved.
    Не может быть раньше даты International freight requested.
    Не может быть позже текущей даты.
  • International freight approved (булево) - дата подтверждения стоимости международной транспортировки компанией Шлюмберже.
    Не может быть раньше даты International freight received.
    Не может быть позже текущей даты.
  • International freight agent (справочник Agents) - агент, который будет проводить международную транспортировку товара.
    Обязательный для заполнения, если заполнена дата International freight requested.
    Не может быть помечен на удаление.
  • POD (справочник Sea and air ports) - point of departure.
    Обязательный для заполнения, если заполнена дата International freight approved.
    Не может быть помечен на удаление.
  • Transitional country (справочник HUBs Countries) - промежуточная страна доставки груза.
    Не может быть помечен на удаление.
  • POA (справочник HUBs Countries) - point of arrival.
    Обязательный для заполнения, если заполнена дата Accepted by specialist.
    Не может быть помечен на удаление.
    При интерактивном выборе система скрывает неподходящие значения.
  • International MOT (справочник MOTs) - способ международной транспортировки.
    Обязательный для заполнения, если заполнена дата International freight approved.
    Не может быть помечен на удаление.
  • International WB (строка) - номер международной накладной.
    Обязательный для заполнения, если заполнена дата ATD.
  • International ETD (дата) - планируемая дата отправки в международную транспортировку.
    Не может быть раньше даты International freight received.
  • International ATD (дата) - фактическая дата отправления груза в международную транспортировку.
    Обязательнй для заполнения, если заполнена дата International ATA.
    Не может быть раньше даты Released from Customs.
    Не может быть позже текущей даты.
  • Pre alert sent (дата) - дата отправки пре-алерта в страну назначения
    Обязательный для заполнения, если заполнены даты International ETA или International ATA.
    Не может быть раньше Released from customs.
    Не может быть позже текущей даты.
  • International ETA (дата) - планируемая дата в страну назначения.
    Не может быть раньше International freight received, International ETD или International ATD.
  • International ATA (дата) - фактическая дата прибытия груза в страну назначения.
    Не может быть раньше International ATD.
    Не может быть позже текущей даты.

Дополнительные реквизиты шапки

  • Created by (справочник Users) - пользователь, создавший заявку на экспорт.
    Заполняется автоматически при первом сохранении заявки на экспорт в базе данных.
  • Creation date (дата) - дата создания заявки на экспорт.
    Заполняется автоматически при первом сохранении заявки на экспорт в базе данных.
  • Modified by (справочник Users) - пользователь, который внес последние изменения в заявку из формы документа. Заполняется автоматически.
  • Modification date (дата) - дата последнего изменения заявки на экспорт из формы документа. Заполняется автоматически.

Таблица 'Other involved locations'

В таблице отражаются TMS involved party locations, которые не указаны ни в одном из предыдущих полей. 

  • Location qualifier (справочник Location qualifiers)
    Обязательный для заполнения реквизит.
  • Location id (текст) - TMS идентификатор локации.
    Система автоматически удаляет строки с пустым Location id.

Таблица 'Other involved contacts'

  • Contact qualifier (справочник Contact qualifiers)
    Обязательный для заполнения реквизит.
  • Contact id (текст) - TMS идентификатор контакта.
    Система автоматически удаляет строки с пустым Contact id.
