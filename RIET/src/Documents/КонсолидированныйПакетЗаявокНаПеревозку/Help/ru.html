Документ "DOC"

Консолидация заказов на доставку

Документ "DOC" входит в одноименную подсистему и предназначен для отражения консолидированных пакетов заказов на доставку грузов.
Таким образом, DOC объединяет Parcels и Invoices, причем при проведении DOC проводятся все Parcels и Invoices, а при отмене DOC отменяются все Parcels и Invoices.

Документ "DOC" вместе с Parcels может быть загружен автоматически из pdf-файла с помощью обработки "DOC loading".

Далее DOCs объединяются в Import shipments.

Мониторинг процесса Green light

Другой функцией документа является отслеживание этапов получения green light.
Выделены следующие этапы: opened, requested, received, granted, booked.
Дата перехода на новый этап отражается в одноименных полях (см. ниже).

Также документ позволяет фиксировать ответственного за нахождения DOC на каждом этапе.
Выделены следующие статусы: opened, pending cca, pending slb, pending hub, red light, closed.
Этапы и статусы жестко связаны, только на этапе requested пользователь может выбрать один из нескольких статусов.

Координатор одновременно отслеживает состояние нескольких DOC.
Чтобы подсказать ему, какие DOC требуют особого внимания предназначен механизм приоретизации (назначения цветов: красного, желтого или зеленого).
Цвет DOC зависит от времени нахождения на том или ином этапе:

  • Для DOC, включенных в поставку, или отмененных DOC устанавливается зеленый цвет
  • Если заполнен реквизит Granted
    • Для авиа и курьерских перевозок желтый цвет устанавливается через 5 дней после даты Granted, а красный - через 10 дней
    • Для остальных видов перевозок желтый цвет устанавливается через 10 дней после даты Granted, а красный - через 20 дней
  • Если заполнен реквизит Requested или Received
    • Для Emeregency грузов желтый цвет устанавливается через 1 день после даты этапа, а красный - через 2
    • Для Urgent грузов желтый цвет устанавливается через 3 дня после даты этапа, а красный - через 6
    • Для остальных грузов желтый цвет устанавливается через 5 дней после даты этапа, а красный - через 10
  • В остальных случаях
    • По-умолчанию устанавливается желтый цвет, красный цвет устанавливается через 24 часа после даты DOC

Система обновляет цвета DOC один раз в день.

Взаимодействие с брокером

Наконец, DOC позволяет автоматизировать взаимодействие таможенного брокера и координатора компании Шлюмберже.

Для принятия в работу DOC брокер нажимает кнопку Accept.

Для уточнения вопросов у координатора брокер нажимает кнопку Return to SLB и оставляет комментарий.

Для выдачи Green light брокер нажимает кнопку Green light, выбирает шаблон почтового сообщения и отправляет его.

Дополнительные возможности

К документу можно прикрепить файлы с помощью кнопки "Attachments".

Ведется история изменения всех значимых реквизитов документа (логи).

Текущие задачи и комментарии можно отражать в поле Current comment, а после выполнения - удалять, при этом удаленный комментарий останется в логах.

Защита от изменения

DOC нельзя изменить после того, как Import shipment доведен до последней стадии.

Структура документа

Основные реквизиты

  • No. (строка) - уникальный номер документа.
    Обязательный для заполнения реквизит.
    Система контролирует уникальность номера.
  • Date (дата) - дата документа.
    Обязательный для заполнения реквизит.
  • Posted (булево) - признак того, что документ проведен, непроведенные документы являются "черновиками".
    Нельзя провести отмененный документ (см. ниже).
  • Cancelled (булево) - признак того, что документ отменен.
    При отмене документа он автоматически становится непроведенным.
    При снятии галочки Cancelled автоматически снимается галочка For deletion, если она была до этого установлена.
  • House keeping (булево)
  • For deletion (булево) - признак того, что документ не просто отменен, а должен быть удален из системы.
    При пометке на удаление автоматически ставится галочка Canceled.
    При снятии пометки на удаление галочка Canceled убирается.
  • Specialist (справочник Users) - импортный специалист, ответственный за обработку документа.
    Заполняется автоматически пользователем, создавшем документ.
  • POD (HUB) (справочник PODs) - point of departure.
    Реквизит должен быть заполнен на этапе выше Opened.
  • CCA (справочник Agents) - customs clearance agent.
    Реквизит должен быть заполнен на этапе выше Opened.
  • Requested POA (справочник Sea and Air Ports) - requested point of arrival.
    Реквизит должен быть заполнен на этапе выше Opened.
  • MOT (справочник MOTs) - mode of transportation.
    Реквизит должен быть заполнен на этапе выше Opened.
  • Urgency (перечисление Urgencies) - срочность груза может быть: Standard, Urgen, Emergency, - и наследуется из PO.
    Emergency может быть назначен только специальным запросом в GOLD.
    Консолидация Urgent происходит на следущий день после получения товара, консолидация Standard происходит на регулярной основе, еженедельно.
    Организация Emergency заказов происходит полностью в ручном режиме.
    Реквизит должен быть заполнен на этапе выше Opened
  • Current comment (строка) - текущий комментарий к DOC.
    Рекомендуется удалять старые комментарии, чтобы они не мешали текущей работе, их история сохранится в логах.

Реквизиты, заполняемые автоматически по данным Invoices

  • Invoice list (строка) - список номеров инвойсов, входящих в данный DOC. Испольуется для удобного нахождения DOC в списке. 
  • PO list (строка) - список PO, входящих в данный DOC. Используется для удобного нахождения DOC в списке.
  • I / E (перечисление Import / export)
    Если в Invoice значение этого реквизита различается - то на этапе выше Opened выдается ошибка.
  • GOLD (булево) - признак того, что это GOLD DOC.
  • Consign-to (справочник Consign-To) - юр. лицо, подразделение, которое будет проводить таможенное оформление товара.
    Если в Invoice значение этого реквизита различается - то на статусе выше Opened выдается ошибка.
  • Seller (справочник Sellers)
    Если в Invoice значение этого реквизита различается - то для GOLD DOC выдается ошибка.

Реквизиты, заполняемые автоматически по данным Parcels

  • Pieces QTY (число) - автоматически заполняемое количество различных Parcel, указанных в DOC.
  • Total Weight (число) - автоматически заполняемая сумма веса брутто в килограммах по всем Parcel'ям.
  • Cubic meters (строка) - автоматически заполняемый объем всех Parcel.

Stages (этапы)

Для отслеживания этапов преданзачены следующие реквизиты шапки документа:

  • Requested (дата) - дата запроса green light у брокера.
    Не может быть раньше даты DOC.
    Автоматически устанавливается статус pending cca, однако пользователь может вручную добавить статус pending slb или red light.
  • First requested (дата) - автоматически заполняемая дата первого запроса green light у брокера.
  • Accepted (дата) - дата принятия DOC на рассмотрение брокером.
    Должен быть предварительно заполнен реквизит requested.
    Не может быть раньше requested.
  • Received (дата) - дата получения green light у брокера.
    Должен быть предварительно заполнен реквизит requested.
    Не может быть раньше requested.
    Не может быть раньше accepted.
    Автоматически устанавливается статус pending slb.
  • Granted (дата) - дата отправки green light на HUB.
    Должен быть предварительно заполнен реквизит received, в противном случае он заполняется автоматически датой granted.
    Не может быть раньше received.
    Автоматически устанавливается статус pending hub.
  • Booked (дата) - дата включения DOC в поставку, заполняется автоматически.
    Должен быть предварительно заполнен реквизит granted (кроме w/o GL DOC).
    Должен быть заполнен при проведении.
  • Current stage (перечисление DOC stages) - текущий этап DOC, заполняется автоматически.
  • W/o GL (булево) - without green light.
    Автоматически заполняется из Shipment, если в списке Out of compliance reasons указана соответствующая причина и не заполнена Granted, или она позже Booked

Statuses (статусы)

Для отслеживания этапов предназначены следующие реквизиты документа:

  • Current status (перечисление DOC statuses) - текущий статус DOC, заполняется автоматически по следующему алгоритму.
    На этапе Open устанавливается статус Open.
    На этапе Requested устанавливается статус Pending CCA, однако если в табличной части Statuses есть строки, то берется статус из самой последней строки.
    На этапе Received устанавливается статус Pending SLB.
    На этапе Granted устанавливается статус Pending HUB.
    На этапе Booked устанавливается статус Closed.
  • Current status date (дата) - дата присвоения текущего статуса.

На этапе Requested пользователь может вручную указать статус в табличной части "Manual statuses", которая имеет следующие реквизиты:

  • Date (дата) - дата присвоения статуса.
    Реквизит должен быть заполнен.
    Не может быть раньше First requested.
    Не может быть позже Received.
    Таблица автоматически сортируется по этому реквизиту.
  • Status (перечисление DOC statuses)
    Реквизит должен быть заполнен.
    Пользователь может указать только один из следующих статусов: pending slb, pending cca, red light.
  • User (справочник Users) - пользователь, установивший статус.
  • Comment (строка) - комментарий к статусу.

Action priority (цвет)

Информация о текущем цвете поставки хранится в следующем реквизите:

  • Action priority (перечисление Статусы важности).

Правила присвоения цвета описаны выше.

Дополнительные реквизиты

  • Creation datе (дата и время) - заполняется автоматически датой создания документа.
  • Last modified (дата и время) - заполняется автоматически датой последнего интерактивного изменения документа.
  • Responsible (справочник Users) - заполняется автоматически пользователем, который внес последние изменения в документ.

Табличная часть Invoices

В этой табличной части перечисляются все Инвойсы, которые относятся к текущему DOC.

  • Invoice (документ Инвойс / Invoice)
    Система автоматически удаляет пустые и повторяющиеся строчки.
    Контролируется, что Invoice указан только в одном DOC. При подборе инвойсов система отображает только инвойсы, не указанные в других DOC.
    Invoice должен быть проведен.
    На этапе Granted для Азербайджана и Туркменистана проверяется заполнение реквизитов Items, таких как Permanent / temporary, Permits required и PSA contract.

Форма документа позволяет просматривать товары, которые относятся к выделенному инвойсу, а также строки парселей, в которых лежат эти товары.

Табличная часть Parcels

В этой табличной части перечисляются все грузовые места, которые относятся к текущему DOC.

  • Parcel (справочник Parcels) - грузовое место.
    Система автоматически удаляет пустые и повторяющиеся строчки.
    Должен быть добавлен хотя бы один Parcel на этапе Requested и выше.
    Контролируется, что Parcel указан только в одном DOC. При подборе система отображает только те грузовые места, которые не указаны в других DOC.
    Parcel не может быть только локальным или отмененным.
    На этапе Granted и всех последующих Parcel должен быть проверен. Система пытается указать соответствующий признак в Parcel автоматически.
    Контролируется, что товары в парселях относятся к текущему DOC.
    Система автоматически заполняет Urgency в грузовых местах.

Форма документа позволяет просматривать также строки парселей.

Система сверяет, что все товары из инвойсов разложены по грузовым местам.

Attached files

К DOC можно прикрепить любые файлы.
Система проверяет, что на этапе Requested прикреплен хотябы один файл к DOC или к инвойсам.
