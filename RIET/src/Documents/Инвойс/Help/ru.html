Документ "Import invoice"

Документ "Invoice" входит в подсистему Import и предназначен для отражения импортного инвойса, на основании которого определяется таможенная стоимость импортируемых товаров.
Экспортные инвойсы отдельно в системе не отражаются, а являются частью документа Export request.

GOLD invoice может быть загружен из pdf-файла с помощью обработки Import invoice loading.
TMS invoice загружается автоматически из TMS.
Остальные инвойсы создаются вручную из документа DOC.

Инвойс может быть распечатан с помощью кнопки "Print".

К документы можно прикреплять любые файлы.

Защита от изменения

Инвойс не может быть изменен после того, как распределена стоимость товаров документом Invoice lines matching или стоимость сервисов документом Services costs allocation.

Реквизиты шапки

  • No. (строка) - номер документа.
    Обязательный для заполнения реквизит.
    В системе не должно существовать двух неотмененных инвойсов с одинаковым номером.
  • Date (дата) - дата документа.
    Обязательный для заполнения реквизит.
  • Posted (булево) - признак того, что документ проведен, непроведенный документ является просто "черновиком".
  • Final (булево) - признак того, что инвойс является финальным (в нем стоят окончательные суммы), а не предварительным, который предоставляется на этапе GL.
    Финальным может быть только проведенный инвойс.
  • Canceled (булево) - признак того, что инвойс отменен.
    При отмене документ автоматически распроводится. 
  • For deletion (булево) - признак того, что инвойс не просто отменен, а будет удален из системы.
    При пометке на удаление автоматически ставится галочка Canceled, а при снятии пометки на удаление - галочка Canceled снимается.
  • Process level (справочник Process levels)
    Обязательный для заполнения реквизит.
  • Specialist (справочник Users) - import specialist.
  • GOLD (булево) - признак того, что инвойс из системы GOLD.
    Инвойс не может быть одновременно GOLD и TMS.
  • TMS (булево) - признак того, что инвойс из системы TMS.
    Инвой не может быть одновременно TMS и GOLD.
  • Seller (справочник Legal entities) - продавец товаров.
    Обязательный для заполнения реквизит.
  • Sold-to (справочник Parent companies) - покупатель товаров.
    Обязательный для заполнения реквизит.
    Страна компании должна совпадать со страной Process level.
    При выборе покупателя система автоматически накладывает необходимые фильтры.
  • Consign-to (справочник Consign-to (imp.) / Shippers (exp.)) - декларант (например, Московское представительство SLI).
    Обязательный для заполнения реквизит.
    Parent company в Consign-to должна совпадать с Sold-to в шапке инвойса.
    При выборе декларанта система автоматически накладывает необходимые фильтры.
  • Incoterms (справочник Incoterms) - условия поставки.
    Обязательный для заполнения реквизит.
  • Contract (справочник Contracts)
    Обязательный для заполнения реквизит (если в Consign-to не указано обратное).
    Consign-to в контракте должен совпадать с Consign-to в шапке инвойса.
    Seller в контракте должен совпадать с Seller в шапке инвойса.
    Incoterms в контракте должен совпадать с Incoterms в шапке инвойса.
    Контракт должен еще действовать на дату инвойса.
    При выборе контракта система автоматически накладывает необходимые фильтры.
  • Operaiting location (справочник Operating locations) - локация, куда должны быть доставлены товары.
    Обязательный для заполнения реквизит (кроме TMS инвойсов, для TMS инвойсов склад доставки узказывается прямо в парселях).
    Страна локации должна совпадать со страной Process level.
    При выборе локации система автоматически акладывает необходимые фильтры.
  • Currency (справочник Валюты) - валюта инвойса.
    Обязательный для заполнения реквизит.
  • DO no. (строка) - номер заявки на доставку.
    Обязательный для заполнения реквизит (в случае, если заявки на доставку нет - используется номер инвойса или номер DOC).
  • Total ex-works (число) - сумма по строкам инвойса.
    Обязательный для заполнения реквизит.
    Сумма, рассчитанная по строкам инвойса, должна совпадать с суммой в этом реквизите.
  • Freight (число) - стоимость транспортировки.
  • Insurance (число) - стоимость страховки.
  • Handling (число) - стоимость хранения.
  • Total Charges (число) - автоматически рассчитываемая сумма Total ex-works + Freight + Insurance + Handling.
  • Skip some checks (булево) - отменяет некоторые проверки при проведении документа.
    Пока только отменяет проверку на совпадение парт-номера в строке инвойса и в строке PO.
  • Transfer (булево) - признак того, что изменился грузополучатель (юр. лицо)
    После установки признака система спросит о необходимости очистить реквизиты No., Sold-to, Consign-to, Incoterms, Contract, Location, AUs and Activities.
  • Created by (справочник Users) - заполняется автоматически пользователем, создавшим документ.
  • Creation datе (дата и время) - заполняется автоматически датой создания документа.
  • Modified by (справочник Users) - заполняется автоматически пользователем, который внес последние изменения в документ.
  • Modification date (дата и время) - заполняется автоматически датой последнего интерактивного изменения документа.
  • Comment (строка) - произвольный пользовательский комментарий.  
  • PO list (строка) - автоматически формируемый список номеров PO для удобного поиска в списке инвойсов.

Таблица Invoice lines (Items)

В таблице Invoice lines (Items) перечисляются товары, которые относятся к данному инвойсу.

Технически данные по товарам хранятся не в самом инвойсе, а в отдельном справочнике Items, описание которого можно посмотреть в соответствующем разделе справки.
