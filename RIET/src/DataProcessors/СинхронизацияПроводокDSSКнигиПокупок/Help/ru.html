Обработка "Синхронизация проводок DSS книги покупок"

Обработка предназначена для загрузки проводок DSS из подсистемы DSS в подсистему Purchase book, а также для выгрузки данных о привязанных документах из подсистемы Purchase book в подсистему DSS.

В подсистему Purchase book загружаются только проводки, у которых модуль = Purchase book.
Если в подсистеме DSS у проводки изменить модуль Purchase book на какой-нибудь другой, то такая проводка будет удалена из подсистемы Purchase book при следующей синхронизации.

Структура проводок DSS в посистеме Purchase book

В подсистеме Purchase book проводки DSS хранятся следующим образом:

  • Для всех проводок, у которых Source code не "AC", не "AP", и не "VP", создается элемент справочника "Проводки DSS деталей счетов книги покупок"
  • Для каждой проводки с Source code = "AC" создается документ Счет книги покупок.
    Такие проводки также называются "Проводки DSS шапки Счетов книги покупок".
    При создании Счета книги покупок система пытается найти и привязать к нему "Проводки DSS деталей счетов книги покупок".
    Алгоритм поиска описан в справке по документу Счет книги покупок.
  • Для каждой проводки с Source code = "AP" или "VP" создается элемент справочника "Проводки DSS Оплат книги покупок".
    Эти проводки потом объединяются в группы, из которых создается документ Оплата книги покупок.
    Подробнее механизм объединения проводок в группы описан в справке к документу Оплата книги покупок.

Операции по синхронизации

Обработка позволяет выполнить следующие операции над каждым типом проводок:

  • Обновить состав проводок:
    • Загрузить новые проводки, которые есть в подсистеме DSS, но нет в подсистеме Purchase book
      ВНИМАНИЕ! Для корректного формирования документов "Счет книги покупок" необходимо вначале загрузить проводки деталей счета книги покупок, а только потом проводки шапки!
    • Удалить старые проводки, которые есть в подсистеме Purchase book, но нет в подсистеме DSS.
      ВНИМАНИЕ! Если такие проводки были привязаны к документам, то система пытается отвязать их от документов, и перезаполнить их табличные части.
      Если отвязать проводку не получается, то она не удаляется из подсистемы.
  • Выгрузить данные о привязанных документах в подсистему DSS

Полная и частичная синхронизация

Обработка позволяет выполнить эти операции для как всех типов проводок (кнопка "Выполнить полную синхронизацию"), так и для каждого типа проводок по-отдельнсти:

  • Проводки "Счетов книги покупок"
    • Проводки шапки Счетов книги покупок
    • Проводки деталей Счета книги покупок
  • Проводки "Оплат книги покупок"

Сведения о количестве проводок

Для каждого типа проводок обработка позволяет оценить количество проводок, которое необходимо обработать.
Для этого надо нажать на соответствующую гиперссылку "Нет данных".
ВНИМАНИЕ! Процесс получения данных может занять некоторе время, в течение которого может показаться, что система "подвисла".
