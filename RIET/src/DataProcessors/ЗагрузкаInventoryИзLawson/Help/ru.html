Обработка "Загрузка inventory из Lawson"

Обработка "Загрузка inventory из Lawson" предназначена для формирования документов Поступление, Перемещение, Списание и Корректировка на основе отчета xr220, выгружаемого из Lawson.
Из отчета загружаются только аналитические и количественные данные.
Суммовые данные в отчете являются управленческими и не подходят для фискального учета, фискальные суммы заполняются с помощью других механизмов.

Для загрузки данных необходимо выбрать файл формата .csv и нажать кнопку Загрузить.

Обработка выделяет порцию проводок с одним IctDocument (номер документа) и IctTransDate (дата документа) и создает из этих проводок один документ.

Вид документа определяется по полю IctDocType:
 - PO - Поступление
 - RT и IT - Перемещение
 - IS и SH - Списание
 - RC, AJ и RA - Корректировка
 - CA и BT игнорируются так как не важны для фискального учета

Прежде чем создать новый документ, система пытается найти его в базе по совпадению комбинации номера и даты.
Если система находит такой документ, выводится сообщение и новый документ не создается, а соответствующая порция проводок игнорируется.
Поэтому, чтобы заново загрузить документ, необходимо в начале пометить старый на удаление и только потом загрузить новый.

В документах заполняются практически все реквизиты, кроме фискальных цен и сумм.
Inventory location ищется по коду. Если локации с необходимым кодом еще нет - система создает новую локацию.
Номенклатура также ищется по коду. Если номенклатуры с необходимым кодом еще нет - система создает новую номенклатуру.
В документе Поступление заполняется PO line по алгоритму, описанному в справке по документу Поступление.

Созданные документы записываются, но не проводятся, так как требуют заполнения фискальных данных.
После заполнения фискальных данных документы можно массово провести с помощью обработки "Восстановление партионного учета Inventory".
